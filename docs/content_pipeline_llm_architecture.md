# Content Pipeline LLM Architecture

## Goal

The content pipeline must extract panel-local visual facts and chart facts from figures and surrounding MinerU evidence. Benchmark metric extraction is allowed only when a valid target scope exists. The main path is source-driven: caption segmentation creates panel tasks before any LLM/VLM call.

## Required input source

Primary input is the original MinerU `*_content_list_v2.json` under the extracted MinerU artifact directory, plus the matching source `layout.json` and image directory.

`structured/content_list_v2_nested.json` is a legacy compatibility artifact and must not be the preferred runtime input.

## Runtime flow

```text
source MinerU content_list_v2.json
→ ContentBlockNormalizer
→ DocumentGraph
→ FigurePanelGraph
→ EvidencePacket
→ local caption-based PanelTask
→ ChartDigitization for chart panels with a valid panel task
→ ImageObservation for non-chart visual facts
→ MetricExtractor LLM/VLM only when panel-local scope exists
→ schema validation / audit
→ ClaimFuser
→ MetricProjector
→ benchmark CSV
```

## Responsibility boundary

### Code responsibilities

Code is responsible for deterministic evidence organization and auditability:

- Parse source MinerU v2 blocks without losing nested content.
- Preserve image/chart paths, bbox, page index, captions, text, tables, formulas, references, and raw block metadata.
- Build DocumentGraph.
- Build FigurePanelGraph from MinerU image/chart blocks and captions.
- Build one EvidencePacket per panel.
- Build panel tasks deterministically from `caption_segment`.
- Mark visual fragments with no panel-local caption task as unusable for image understanding and digitization.
- Validate strict LLM phases against a schema.
- Fail the run if required LLM phases are unavailable or invalid.
- Project accepted structured metric candidates to CSV.

### LLM/VLM responsibilities

LLM/VLM is responsible only for visual understanding inside the task boundaries created by code:

1. Chart and image visual phases
   - Extract chart facts, chart points, and non-chart visual fact candidates without paper-level ontology guidance.
   - Do not decide benchmark target groups or publishable metric permissions.

2. MetricExtractor
   - Extract benchmark metric candidates using the image, EvidencePacket, and PanelSemanticProfile.
   - Output structured metrics only, not free-form visual descriptions.
   - Attach evidence text and evidence level to every metric.
   - This phase is skipped unless panel-local target scope is valid.

## No fallback rule

If `model_client` is missing, the run must fail.

If MetricExtractor fails for a panel, that panel must be marked failed. With `fail_fast=True`, the full run fails.

Rule-based PanelProfiler and rule-based extractor behavior are not valid production fallbacks for the required LLM pipeline.

## Output contract

The audit output must include:

- `panel_semantic_results` generated from local caption-based panel tasks
- `extractor_results` generated by MetricExtractor
- LLM phase audit traces
- metric rows and rejected metric rows

## Acceptance criteria

Using the real 003 MinerU source v2 fixture:

- The CLI accepts `extracted/*_content_list_v2.json` as input.
- FigurePanelGraph has 5 figures and 30 panels.
- No synthetic `figure-30*` figures are produced.
- The run does not call PanelSemanticClassifier or ChartVisualSemantic on the main path.
- The run calls ChartDigitization only for chart panels with a valid panel-local caption task.
- The run calls MetricExtractor only when panel-local target scope exists.
- Without an LLM client, the strict pipeline fails instead of silently falling back.
