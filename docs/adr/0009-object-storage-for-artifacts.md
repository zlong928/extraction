# ADR-009: 对象存储保存大型与非结构化产物

- 状态：已采纳
- 日期：2026-07-11

## 背景

PDF、数十万图片、MinerU 目录、模型原始响应和交付文件会快速超过共享目录与数据库 BLOB 的合理边界。API 与 worker 也不能依赖同一宿主机绝对路径。

## 决策

所有持久字节通过 `StorageAdapter` 访问。开发和测试使用 `LocalStorageAdapter`，生产使用 `S3StorageAdapter`（兼容 AWS S3、MinIO 等）。PostgreSQL 的 `storage_objects` 只记录对象键、URI、SHA-256、大小、媒体类型、ETag 和关系元数据。

对象键使用条件创建：本地实现用同文件系统原子 link，S3 实现用 `If-None-Match: *`。键已存在时必须校验 SHA-256 和大小，相同内容复用，不同内容拒绝，避免并发 writer 把数据库校验值与对象字节写成两个版本。业务通过对象 ID/键恢复字节；需要本地路径的 MinerU/模型库只在临时工作区 materialize，并在完成后把产物上传回对象存储。

## 结果

- API/worker 可跨主机运行，不再需要共享持久目录。
- 删除论文采用业务软删除，避免破坏运行审计和已发布交付。
- 对象生命周期、版本控制、加密、跨区复制与垃圾回收由部署策略管理；清理必须先证明对象没有运行或交付引用。
